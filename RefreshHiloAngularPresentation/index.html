What is Angular?

“AngularJS is a structural framework for dynamic web apps. It lets you use HTML as your template language and lets you extend HTML's syntax to express your application's components clearly and succinctly. Angular's data binding and dependency injection eliminate much of the code you would otherwise have to write. And it all happens within the browser, making it an ideal partner with any server technology.”

What does this mean?

  • It's a framework instead of a library
  • HTML is your template language
  • It extends the HTML syntax
  • Provides data binding
  • Dependency injection is a first class citizen
  • It is all rendered on the client
  • And it is much more than what is just loaded in that paragraph...

Directives

  • Directives are Angular's way of extending and bending the HTML
  • Directives allow for Angular's "HTML Compiler" to attache specific behaviors to your application
  • Examples
    • ngBind - {{}}
    • ngApp
    • ngController
    • ngClick
    • ngView
    • ngRepeat

Expressions & Filters

  • Expressions
    • JavaScript-like code snippets that are used in ngBind or {{}}
    • More forgiving than JavaScript, won't throw an exception when a value is undefined
    • Cannot use conditionals, loops, or exceptions
  • Filters
    • Used to format an expression for user display
    • Examples
      • orderBy
      • currency
      • date

Scope

  • Scope is the glue that ties the data in the controller to the view
  • A lot of people refer to this as the ViewModel as well
  • In code, it is referenced as $scope
  • Access of variables are similar to how a variable scope works in server side code, but with some "gotchas"
  • Provide APIs ($watch) to observe model mutations
  • Provide APIs ($apply) to propogate model changes through the system

Controllers

  • A JavaScript contructor function that is used to augment the scope
  • Created each time the controller comes into use
  • Controllers should be used to set up the initial state and add behavior
  • Do not use them for:
    • DOM Manipulation - this should be done using Directives
    • Format input - Use form controls instead
    • Filter output - Use filters instead
    • Share code or state - Use services instead

Modules

  • The container for the different parts of the app (controllers, services, directives, etc)
  • A module is created by using angular.module('myApp', []) - [] are used to inject any modules that your module depends on
  • A module can be referenced within the app with angular.module('myApp') - notice no []

Forms & Form Validation

  • Just like in standard HTML, this is just a way for users to input data
  • Difference when using AngularJS is the fact you get two way data binding on your form fields
  • AngularJS also provides validation services
  • ng-messages and ng-message are two new directives added in AngularJS 1.3 that make form validation easier - requires angular-messages.js

Forms & Form Validation (cont'd)

  • Adds CSS classes to form fields depending on the current state of the field
    • ng-invalid - the model is invalid
    • ng-valid - the model is valid
    • ng-pristine - the control hasn't been interacted with yet
    • ng-dirty - the control has been interacted with
    • ng-touched - the control has been blurred
    • ng-untouched - the control hasn't been blurred
    • ng-pending - any async validators are unfulfilled
  • The above css classes are also properties on the model where ng- is replaced with $

Routing

  • Uses ngView to determine where to place the template
  • Defined on a the $routeProvider using when and otherwise methods
  • At a minimum, you need to specify the route and the template
  • Accessed via #/<route>
  • Requires that angular-route.js is loaded as well

Services

  • Substituable objects that are wired together using dependency injection
  • Lazily instantiated - will only be instantiated when an application component depends on it
  • Singletons - each component gets a reference to the single instance generated by the service factory
